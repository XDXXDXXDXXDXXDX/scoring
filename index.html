# 介绍

快应用性能优化实践文档，旨在通过展现一个快应用的优化过程，给开发者一个直观的优化思路和方法。开发者可以仿照优化过程，使用示例中用到的工具和方法对自己的快应用快速进行性能优化。

优化的方法主要分成运行时优化和 rpk 包大小优化，每一块都有几个具体的优化手段供您参考。

# rpk 包大小优化

rpk包即开发者完成快应用开发时执行hap build/hap release打包出来的程序包，用户在运行快应用时，快应用引擎会将包下载到手机中运行。所以通过减小包体积的大小，可以减少引擎下载包的时间，让引擎更早的去解析包并进行渲染。

# 运行时优化

运行时即快应用引擎解析rpk包后执行应用代码的时间。这一阶段的优化主要是减少或者推迟不必要的代码执行和渲染，尽快的将用户可视的部分呈现给用户。
