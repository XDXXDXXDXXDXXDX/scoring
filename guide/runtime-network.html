# 网络耗时优化

网络耗时会对首屏的渲染时间和用户的体验有较大的影响，因此需要尽量较少网络请求所消耗的时间，给用户更好的体验

# 避免网络接口请求异常

网络接口请求失败会导致快应用某些页面无法正常展示，因此应该保证所有接口都能请求成功

# 减少首屏渲染时的网络请求数

网络请求耗时会增加首屏渲染的时间，因此可以减少首次渲染时的网络请求，具体请参考dom 优化

# 数据缓存

可以将请求的数据进行数据缓存，在下次进行数据请求的时候判断是否存在数据请求，若存在则直接从缓存中读取，无需再次进行网络请求，可大大提升加载速度

# 减少图片的大小

图片太大会增加下载时间和内存的消耗，因此需要合理控制图片的大小， 具体可参考减少图片大小文档

# 减少图片请求数目

短时间内发起太多图片请求会触发浏览器并行加载的限制，可能导致图片加载慢，用户一直处理等待。应该合理控制数量，可考虑使用雪碧图技术或在屏幕外的图片使用懒加载
