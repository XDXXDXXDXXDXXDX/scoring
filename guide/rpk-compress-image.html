# 图片大小

一般快应用中，图片会占大部分的体积，而图片太大会增加下载时间和内存的消耗，所以有必要对图片大小进行优化

# 压缩图片

首先对所有用到的图片，不论是本地的资源亦或是从服务端请求的资源都应该进行压缩。

推荐在tinypng (opens new window)中对图片进行压缩。

# 其他

前端常用的优化手段在快应用中也可以使用

  • 对比较大的图片可以托管到优质的cdn上，减小包的体积并获得更快的加载速度
  • 即使压缩了图片也要注意图片大小，例如图片的大小不要比其实际需要展示的大小大太多（例如实际展示的大小为200px*100px，那么图片大小也在200px*100px左右即可），单张图片尽量控制在200KB以内
